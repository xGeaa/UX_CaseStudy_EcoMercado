# Parte II: Caso de estudio — EcoMercado UGR

> Análisis UX, propuesta de diseño y autoevaluación 

---

## Descripción

Esta segunda parte del trabajo final aplica las técnicas y metodologías de diseño centrado en el usuario (DCU) a un caso real: el **EcoMercado UGR**, una iniciativa de la Universidad de Granada y la Red Agroecológica de Granada que acerca la producción agroecológica local a la comunidad universitaria.

El proceso seguido consta de tres fases:

1. **Análisis experto** de una web de mercado ecológico real como referente: [Huerta Madrid (nuestrashuertas.com)](https://www.nuestrashuertas.com/), evaluando su usabilidad, accesibilidad, experiencia de usuario y adaptación a dispositivos móviles.
2. **Propuesta de diseño** para una app del EcoMercado UGR, basada en los insights extraídos del análisis anterior y en las necesidades del usuario universitario.
3. **Autoevaluación** del proceso, conectando lo realizado con las prácticas de la asignatura.

Todo el análisis se realiza con criterios de experto, aplicando herramientas y metodologías estudiadas en DIU: heurísticas de Nielsen, pautas WCAG 2.1, definición de Personas, Empathy Map, ScopeCanvas, arquitectura de información y wireframing.

## Metodología aplicada

| Herramienta/Técnica | Fase | Descripción |
|---|---|---|
| Personas | Análisis + Propuesta | Definición de perfiles de usuario de Huerta Madrid y del EcoMercado UGR |
| Evaluación heurística (Nielsen, 1994) | Análisis | Revisión de las 10 heurísticas sobre la web de Huerta Madrid |
| Usability Review | Análisis | Puntuación numérica de usabilidad con checklist |
| WCAG 2.1 (W3C, 2018) | Análisis | Evaluación de accesibilidad (niveles A y AA) |
| Análisis responsive | Análisis | Comparativa escritorio vs. móvil |
| Empathy Map | Propuesta | Síntesis de hallazgos e insights |
| ScopeCanvas | Propuesta | Definición del alcance y propuesta de valor |
| Arquitectura de información | Propuesta | Sitemap y User Flow de la app |
| Wireframing | Propuesta | Bocetos de las pantallas principales |
